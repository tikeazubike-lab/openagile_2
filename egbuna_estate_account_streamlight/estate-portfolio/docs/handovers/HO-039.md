### Analysis

All hits are connected to the same feature: **inline row editing of holding
data** (`num_shares` and `avg_purchase_price`). The flow is:

1. Admin clicks Pencil icon in Actions column → `handleEditClick` sets `editingRowId`
2. Table row renders `InlineEditRow` component (a child component with its own state for edit inputs)
3. User edits `num_shares` and `avg_purchase_price` fields inline
4. Save → `handleSaveInline` calls `updateHolding.mutateAsync` → `setEditingRowId(null)`
5. Cancel → `setEditingRowId(null)`

**Ruling request:** This is inline data-write editing of fields, not modal-based editing.
HO-023 states "All inline editing removed. All CRUD activities moved to Admin section."
If Claude rules this a violation, the fix would be to remove the Pencil icon,
`editingRowId` state, `handleEditClick`/`handleSaveInline` functions, and the
`InlineEditRow` component integration from `_app.holdings.tsx`, replacing with
modal-based editing inside the settings section.

`settings.users.tsx` was also checked — its `isEditing` is modal-based (Edit user
modal opens, no inline editing). Acceptable per HO-038 ruling.
