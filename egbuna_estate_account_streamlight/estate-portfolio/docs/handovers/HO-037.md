## 4. Failure Analysis

### Failure 1: AT-004-A03 + AT-004-D02 — SUPERADMIN excluded from admin

**Root cause:** Both the backend `require_admin` dependency and the frontend `isAdmin()` auth store check only compare `role === "admin"`. The "superadmin" role is excluded.

```
# backend/app/deps.py → require_admin
if current_user.role != "admin": raise HTTPException(403, "Admin access required")

# frontend/authStore.ts → isAdmin
return get().user?.role === "admin"
```

**Fix direction:** Change both checks to allow role in ["admin", "superadmin"]. This is a one-line backend change + one-line frontend change.

**Ownership:** Should be included in HO-024 remediation or filed as a separate fix before F-016.

### Failure 2: AT-004-B04 — Residual isEditing in holdings.tsx

**Root cause:** The holdings page component (`_app.holdings.tsx`) still contains an `isEditing` local variable at line 307 used for inline row selection (`editingRowId`). The `editMode` toggle concept (the togglable mode) is eliminated (0 grep hits), but row-level inline editing state remains.

**Fix direction:** Remove inline editing from holdings.tsx, replace with modal-based edits. This is a more involved frontend change.

**Ownership:** HO-024 scope was editMode toggle removal. Inline row editing may be intentional. Confirm with PO if holdings need modal-only editing.
