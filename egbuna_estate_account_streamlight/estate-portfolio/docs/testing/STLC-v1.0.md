# [TC] STLC-1.0 — Software Testing Life Cycle

## Purpose
Define the process for verifying features before they are merged to `test` and `main`.

## Process
1. Test Plan creation (`TC-XXX`) based on Business Requirements (`BR-XXX`).
2. Execution of test cases in the staging environment.
3. Filing of Acceptance Test results (`AT-XXX`) detailing what passed, failed, or was skipped.
4. Logging of bugs for any failures.
