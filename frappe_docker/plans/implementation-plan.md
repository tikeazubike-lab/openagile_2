# Implementation Plan: Integrating tutor-connect-hub with Frappe Backend

## Executive Summary

This document outlines the step-by-step implementation plan for integrating the existing React 18 frontend (`tutor-connect-hub/`) with the Frappe/ERPNext backend for the Tutor Marketplace application. The frontend was generated by Lovable AI and uses React 18, TypeScript, Vite, Tailwind CSS, and shadcn-ui components.

## Overview

### Frontend Status
The existing frontend (`tutor-connect-hub/`) includes:
- ✅ React 18.3.1 with TypeScript 5.6.3
- ✅ Vite 6.0.3 build tool
- ✅ Tailwind CSS 4.0.0 with custom design system
- ✅ shadcn-ui components (Radix UI primitives)
- ✅ TanStack Query 5.62.7 for server state management
- ✅ React Hook Form 7.54.2 with Zod 3.24.1 for form validation
- ✅ React Router DOM 7.1.1 for routing
- ✅ Framer Motion 11.15.0 for animations
- ✅ Recharts 2.15.0 for charts
- ✅ date-fns 4.1.0 for date handling
- ✅ Lucide React 0.468.0 for icons
- ✅ Landing page with Hero, Stats, Subjects, How It Works, Testimonials, CTA sections
- ✅ Header and Footer components
- ✅ Design system with custom color palette

### Backend Status
The Frappe backend needs to be created:
- ❌ New Frappe app `tutor_marketplace` - NOT CREATED
- ❌ New site `tuts.erpnext.zubbystudio.shop` - NOT CREATED
- ❌ DocTypes for Tutor Marketplace - NOT CREATED
- ❌ API endpoints - NOT CREATED
- ❌ Payment gateway integrations - NOT CREATED

## Implementation Strategy

The implementation will follow a phased approach, with the frontend and backend teams working in parallel where possible. The frontend will use mock data during development until the backend APIs are available.

## Phase 1: Backend Foundation (Week 1)

### Backend Tasks

1. **Create New Frappe App**
   - Create `tutor_marketplace` Frappe app
   - Set up app structure
   - Configure app hooks
   - Create app icon and metadata

2. **Create New Site**
   - Create new site `tuts.erpnext.zubbystudio.shop`
   - Configure site settings
   - Install `tutor_marketplace` app on the site
   - Configure site for multi-tenancy

3. **Set Up Docker Configuration**
   - Update `docker-compose.yml` to include new site
   - Configure Traefik routing for `tuts.erpnext.zubbystudio.shop`
   - Set up volume mounts for new site
   - Test container startup

4. **Create Core DocTypes**
   - Create Tutor Profile DocType
   - Create Student Profile DocType
   - Create Session Schedule DocType
   - Create Payment Transaction DocType
   - Create Tutor Availability DocType
   - Create Tutor Review DocType
   - Create Marketplace Settings DocType

5. **Configure CORS**
   - Enable CORS for frontend access
   - Configure allowed origins
   - Test CORS configuration

### Frontend Tasks

1. **Install frappe-js-sdk**
   ```bash
   cd tutor-connect-hub
   npm install frappe-js-sdk
   ```

2. **Create API Client Module**
   - Create `src/lib/api-client.ts`
   - Initialize frappe-js-sdk client
   - Create API methods for all DocTypes
   - Create custom API methods (search, profile, etc.)

3. **Create Environment Variables**
   - Create `.env` file with:
     ```
     VITE_FRAPPE_URL=https://tuts.erpnext.zubbystudio.shop
     VITE_STRIPE_PUBLIC_KEY=your_stripe_public_key
     VITE_PAYPAL_CLIENT_ID=your_paypal_client_id
     ```

4. **Update Vite Configuration**
   - Update `vite.config.ts` to proxy API requests
   - Configure development server settings

5. **Create Mock Data**
   - Create `src/mocks/` directory
   - Create mock data files for all API endpoints
   - Create mock data toggle utility

6. **Update App.tsx**
   - Add QueryClient configuration
   - Set up base structure for AuthProvider

## Communication Protocol

### Frontend-Backend Communication

1. **API Contract**
   - Frontend and backend teams should agree on API contracts
   - Document all API endpoints with request/response formats
   - Version API endpoints

2. **Issue Tracking**
   - Create GitHub issues for backend API requests
   - Tag issues with `frontend`, `backend`, or `both`
   - Assign appropriate team members

3. **Regular Syncs**
   - Schedule weekly sync meetings
   - Discuss progress and blockers
   - Plan upcoming work

### Mock Data Strategy

1. **When to Use Mock Data**
   - Backend API is not yet implemented
   - Backend API is broken or unavailable
   - Testing frontend without backend dependency

2. **Mock Data Management**
   - Store mock data in `src/mocks/` directory
   - Use environment variable to toggle mocks
   - Ensure mock data matches API response structure

3. **Transition to Real Data**
   - Gradually replace mock data with real API calls
   - Test each API endpoint before switching
   - Keep mock data for fallback
